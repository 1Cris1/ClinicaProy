🩺 HISTORIA DE USUARIO 3 y 4: Evolucion Medica y Receta Digital
---------------------------------------------------------------------------

Esta funcionalidad permite al medico registrar sus notas clinicas (evolucion)
y generar una receta digital con medicamentos al finalizar la atencion.

[Detalle linea por linea]:
* Lineas 127 y 128: [@PostMapping y @Transactional]
  -> @PostMapping indica que la web enviara los datos del formulario por el metodo seguro POST.
  -> @Transactional es el "Todo o Nada". Si ocurre un error guardando un medicamento de la receta, 
     cancela la operacion completa y no guarda nada (ni siquiera las notas) para evitar que la 
     base de datos quede inconsistente.
* Lineas 129 a 136: [public String completarConsulta(...)]
  -> Se reciben todos los campos del formulario web. Destaca "med_nombre" que es una "List<String>", 
     lo que significa que el formulario puede enviar multiples medicamentos a la vez.
* Linea 138 y 139: [Medico medico = getMedicoAutenticado(principal);]
  -> Verifica por seguridad (usando Spring Security) quien es el medico que esta conectado en el sistema.
* Linea 141 y 142: [Cita cita = citaRepository.findById...]
  -> Busca la cita y valida que exista y que pertenezca exactamente a ese medico (seguridad para que 
     un medico no pueda alterar la cita de otro doctor).
* Lineas 143 a 147: [cita.setEstado("Completada"); ... citaRepository.save(cita);]
  -> Esta es la **HU 3 (Evolucion)**. Se actualiza el estado, se guardan las notas medicas escritas 
     por el doctor y se manda el comando UPDATE a la base de datos.
* Linea 149: [if (medNombres != null && !medNombres.isEmpty()) {]
  -> Esta es la **HU 4 (Receta)**. Si el doctor agrego al menos un medicamento en la pantalla, 
     se inicia la creacion de la receta.
* Lineas 150 a 156: [Receta receta = Receta.builder()...]
  -> Se usa el patron Builder para crear el "cabezal" de la receta, generandole un codigo unico 
     basado en el tiempo actual (milisegundos) y vinculandolo al paciente y al medico de la cita.
* Linea 158: [receta = recetaRepository.save(receta);]
  -> Se guarda la cabecera de la receta en la base de datos para obtener su ID autogenerado.
* Lineas 159 a 171: [for (int i = 0; i < medNombres.size(); i++) { ...]
  -> Un bucle recorre todos los medicamentos agregados en el Frontend. Por cada uno, crea un objeto 
     "MedicamentoPrescrito" (el detalle de la receta), lo vincula a la receta principal y lo guarda.
* Linea 173: [log.info(...)]
  -> Deja un registro en el archivo de auditoria (consola/logs) del servidor indicando que este medico 
     ha finalizado su consulta con exito.
* Linea 175: [return "redirect:/medico/inicio?exito";]
  -> Devuelve al medico a su panel principal mostrando el aviso de exito.

[Detalle linea por linea]:
* Linea 145: [<form th:action="@{/medico/consulta/completar}" method="post">]
  -> Inicia el gran formulario que envuelve toda la consulta. Al enviar, apunta al Backend 
     que acabamos de explicar.
* Linea 146: [<input type="hidden" name="idCita"...>]
  -> Oculto a la vista del medico, pero fundamental: envia el ID de la cita actual para 
     que el backend sepa que cita se esta completando.
* Linea 184: [<textarea name="notasMedicas"...>]
  -> El cuadro de texto principal donde el medico escribe la evolucion. El atributo "required" 
     obliga a que el medico escriba algo antes de poder guardar.
* Linea 280: [<template id="med-item-template">]
  -> Una etiqueta <template> nativa de HTML5. Su contenido es invisible al cargar la pagina, 
     pero sirve como "molde" para que JavaScript clone y pegue cajas de medicamentos al hacer 
     clic en "Agregar Medicamento".
* Linea 284: [<input type="text" name="med_nombre"... onkeyup="buscarProducto(this)">]
  -> Campo para el nombre del medicamento. El evento "onkeyup" dispara una busqueda en 
     tiempo real en la farmacia (API) cada vez que el medico tipea una letra.


3. IMPACTO EN BASE DE DATOS (Las tablas afectadas)
---------------------------------------------------------------------------
Cuando este codigo se ejecuta, ocurren 3 cosas en la BD:
1. Tabla `citas`: Se hace un UPDATE de `estado` a "Completada" y se guarda el texto en `notas_medicas`.
2. Tabla `recetas`: Se hace un INSERT con los datos base de la receta (ID paciente, ID medico).
3. Tabla `medicamentos_prescritos`: Se hace un INSERT multiple, agregando una fila por 
   cada medicamento, dosis y frecuencia que el medico receto.
